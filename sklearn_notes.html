01
NumPy
Fast numerical computing — the backbone of ML data handling
ℹ️ NumPy arrays are stored in contiguous memory blocks, making operations up to 50x faster than Python lists. Every ML library (pandas, sklearn, tensorflow) uses NumPy under the hood.

📦 Why NumPy over Python Lists?

  • No Python loops needed — vectorized ops
  • Fixed data type = faster computation
  • Broadcasting (operations on different shapes)
  • Built-in math: mean, std, dot product, etc.
  • Basis for Pandas DataFrames and sklearn inputs

🔑 Key Attributes of ndarray

  • arr.shape → dimensions (rows, cols)
  • arr.ndim → number of dimensions
  • arr.dtype → data type (int32, float64…)
  • arr.size → total number of elements
  • arr.T → transpose
import numpy as np

# ── Creating arrays ──────────────────────────
arr1d = np.array([1, 2, 3, 4])
arr2d = np.array([[1,2,3], [4,5,6]])  # 2 rows × 3 cols
zeros  = np.zeros((3,4))           # 3×4 matrix of 0s
ones   = np.ones((2,2))            # 2×2 matrix of 1s
eye    = np.eye(3)                 # 3×3 identity matrix
rng    = np.arange(0, 10, 2)       # [0, 2, 4, 6, 8]
lin    = np.linspace(0, 1, 5)      # 5 evenly spaced between 0-1

# ── Random ───────────────────────────────────
np.random.seed(42)               # reproducibility
rand  = np.random.rand(3, 3)     # uniform [0,1)
randn = np.random.randn(100)     # standard normal distribution
randi = np.random.randint(1, 10, (4,4))  # integers between 1-10

# ── Reshaping ────────────────────────────────
a = np.arange(12)
b = a.reshape(3, 4)   # 3×4 — total elements must match
c = b.flatten()         # back to 1D
d = a.reshape(-1, 1)  # column vector (sklearn often needs this)

# ── Math operations (vectorized) ─────────────
x = np.array([1.0, 2.0, 3.0])
print(x.mean(), x.std(), x.sum(), x.min(), x.max())
print(np.dot(x, x))         # dot product = 14.0
print(np.sqrt(x))           # element-wise square root
print(np.log(x))            # natural log

# ── Indexing & slicing ───────────────────────
arr = np.arange(10)
print(arr[2:6])              # [2, 3, 4, 5]
print(arr[arr > 5])          # boolean mask → [6,7,8,9]

m = np.array([[1,2,3],[4,5,6]])
print(m[0, :])               # first row: [1,2,3]
print(m[:, 1])               # second column: [2,5]

# ── Broadcasting ─────────────────────────────
A = np.ones((3, 3))
v = np.array([1, 2, 3])
print(A + v)  # v added to each row of A — no loops!

Pandas
Data analysis and manipulation — your main tool for real-world datasets
ℹ️ A DataFrame is a 2D labeled table (like an Excel sheet). A Series is a single column. Every ML project starts here — load, explore, and clean your data with Pandas before feeding it to sklearn. 
import pandas as pd

    'Age'   : [24, 30, 22, 28],
    'Salary': [50000, 80000, 45000, 70000],
    'Gender': ['F', 'M', 'M', 'F']
}
df = pd.DataFrame(data)

# ── Read / write files ───────────────────────
df = pd.read_csv('data.csv')
df = pd.read_excel('data.xlsx')
df.to_csv('output.csv', index=False)

# ── First look ───────────────────────────────
df.head(5)          # first 5 rows
df.tail(3)          # last 3 rows
df.info()           # dtypes + non-null counts
df.describe()       # stats: mean, std, min, 25%, 50%, 75%, max

Data Preprocessing
Garbage in → garbage out. Clean, encode, and scale your data
⚠️ Golden Rule: Always fit scalers and encoders on training data only. Then use transform() on test data. Fitting on test data causes data leakage — your model cheats by seeing test statistics.

🔍 Missing Value Strategies

StrategyWhen to useCode
Mean imputationNumeric, no outliersfillna(df['col'].mean())
Median imputationNumeric, with outliersfillna(df['col'].median())
Mode imputationCategorical columnsfillna(df['col'].mode()[0])
Drop rowsVery few missing (<5%)df.dropna()
Drop columnColumn has >50% missingdf.drop('col', axis=1)
KNN / model-basedComplex relationshipssklearn KNNImputer
from sklearn.impute import SimpleImputer, KNNImputer

# ── Check missing values ─────────────────────
df.isnull().sum()
df.isnull().mean() * 100   # % missing per column

# ── Pandas fill ──────────────────────────────
df['Age']      = df['Age'].fillna(df['Age'].median())
df['Embarked'] = df['Embarked'].fillna(df['Embarked'].mode()[0])

# ── Sklearn imputer (better for pipelines) ───
imputer = SimpleImputer(strategy='median')   # mean/median/most_frequent/constant
X_imp   = imputer.fit_transform(X)

# ── KNN imputer: uses similar rows to fill ───
knn_imp = KNNImputer(n_neighbors=5)
X_imp   = knn_imp.fit_transform(X)

🏷️ Label Encoding

Converts category to integer. Use for binary or ordinal features only. Risk: model may assume order (0 < 1 < 2).

from sklearn.preprocessing import LabelEncoder
le = LabelEncoder()
df['Gender'] = le.fit_transform(df['Gender'])
# Male→0, Female→1

🔢 One-Hot Encoding

Creates binary column per category. Use for nominal features with no order. Beware: adds many columns for high-cardinality.

# Pandas way (simple)
df = pd.get_dummies(df, columns=['City'])

# Sklearn way (for pipelines)
from sklearn.preprocessing import OneHotEncoder
ohe = OneHotEncoder(drop='first')

📊 Ordinal Encoding

Use when categories have a meaningful order: Small < Medium < Large. You define the order explicitly.

from sklearn.preprocessing import OrdinalEncoder
oe = OrdinalEncoder(
  categories=[['Low','Med','High']]
)
df['Size'] = oe.fit_transform(df[['Size']])

📐 StandardScaler (Z-score)

z = (x − μ) / σ

Mean becomes 0, std becomes 1. Best for algorithms that assume Gaussian distribution: Linear Regression, Logistic Regression, SVM, PCA.

from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)  # NOT fit_transform!

📏 MinMaxScaler (Normalization)

x' = (x − min) / (max − min)

Values become 0–1. Best for Neural Networks, KNN, image data. Sensitive to outliers (they push other values toward 0).

from sklearn.preprocessing import MinMaxScaler
scaler = MinMaxScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)
# ── Train-Test Split ─────────────────────────
from sklearn.model_selection import train_test_split

X = df.drop('target', axis=1)
y = df['target']

X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,      # 20% test, 80% train
    random_state=42,   # reproducibility
    stratify=y          # preserves class ratio (for classification)
)

Linear Regression
Predict continuous values by fitting a straight line
y = β₀ + β₁x₁ + β₂x₂ + … + βₙxₙ + ε

📖 How It Works

  • Finds the best-fit line minimizing Sum of Squared Errors (SSE)
  • Uses Ordinary Least Squares (OLS) or Gradient Descent
  • β coefficients: how much y changes per unit of each x
  • β₀ (intercept): value of y when all x = 0

⚠️ Assumptions

  • Linear relationship between X and y
  • Residuals are normally distributed
  • No multicollinearity (features not correlated with each other)
  • Homoscedasticity: constant variance of errors
from sklearn.linear_model import LinearRegression
from sklearn.preprocessing import PolynomialFeatures

# ── Simple Linear Regression ─────────────────
model = LinearRegression()
model.fit(X_train, y_train)
pred  = model.predict(X_test)

print("Intercept:", model.intercept_)
print("Coefficients:", model.coef_)
# coef_[i] = effect of feature i on y (positive = increases, negative = decreases)

# ── Polynomial Regression (curved fit) ───────
poly    = PolynomialFeatures(degree=2)  # adds x², x₁x₂, etc.
X_poly  = poly.fit_transform(X_train)
model_p = LinearRegression()
model_p.fit(X_poly, y_train)

Logistic Regression
Classification using probabilities and the sigmoid function
ℹ️ Despite the name, Logistic Regression is a classification algorithm. It outputs a probability (0–1), then classifies using a threshold (default 0.5).
P(y=1) = 1 / (1 + e^−z) where z = β₀ + β₁x₁ + … + βₙxₙ
from sklearn.linear_model import LogisticRegression

model = LogisticRegression(
    C=1.0,              # regularization (smaller C = more regularization)
    solver='lbfgs',    # optimizer: lbfgs, liblinear, saga
    max_iter=200,      # increase if convergence warning
    multi_class='auto' # handles multi-class automatically
)
model.fit(X_train, y_train)

pred       = model.predict(X_test)          # class labels [0,1]
pred_proba = model.predict_proba(X_test)   # probability [0.3, 0.7]
# Change threshold: pred = (pred_proba[:,1] >= 0.4).astype(int)

K-Nearest Neighbors (KNN)
Classify/predict using the K most similar training examples

🧭 How It Works

  • No training phase — it's a lazy learner
  • For each test point, find K nearest training points
  • Classification: majority vote of K neighbors
  • Regression: average of K neighbors' values
  • Distance: usually Euclidean — so scale your features!

🎯 Choosing K

  • K=1: overfitting, too sensitive to noise
  • Large K: underfitting, too smooth
  • Best practice: try odd K values (1,3,5,7…) and use elbow method
  • Rule of thumb: K ≈ √(n_samples)
from sklearn.neighbors import KNeighborsClassifier

# ── Find best K ──────────────────────────────
error_rates = []
for k in range(1, 20):
    knn = KNeighborsClassifier(n_neighbors=k)
    knn.fit(X_train, y_train)
    error_rates.append(1 - knn.score(X_test, y_test))
# plot error_rates and pick the elbow

model = KNeighborsClassifier(
    n_neighbors=5,
    metric='euclidean',   # or 'manhattan', 'minkowski'
    weights='distance'    # closer neighbors get more vote weight
)
model.fit(X_train, y_train)

Decision Tree
Flowchart-like model — splits data based on feature thresholds

📐 Key Concepts

  • Gini Impurity: how mixed the classes are in a node (0 = pure)
  • Entropy / Information Gain: how much info a split provides
  • max_depth: limits tree size → prevents overfitting
  • min_samples_split: minimum samples needed to split a node

✅ Pros / ❌ Cons

  • ✅ No feature scaling needed
  • ✅ Handles mixed data types
  • ✅ Highly interpretable — you can visualize the tree
  • ❌ Prone to overfitting on training data
  • ❌ Unstable — small data changes = very different tree
from sklearn.tree import DecisionTreeClassifier, plot_tree

model = DecisionTreeClassifier(
    criterion='gini',      # or 'entropy'
    max_depth=4,           # limit depth to prevent overfitting
    min_samples_split=10,  # need 10+ samples to split
    random_state=42
)
model.fit(X_train, y_train)

# ── Visualize the tree ───────────────────────
plt.figure(figsize=(20,10))
plot_tree(model, feature_names=X.columns,
          class_names=['0','1'], filled=True)

# ── Feature importance ───────────────────────
importance = pd.Series(model.feature_importances_, index=X.columns)
importance.sort_values().plot(kind='barh')

Random Forest
Ensemble of decision trees — powerful, accurate, and robust
💡 Random Forest uses two sources of randomness: Bootstrap sampling (each tree trained on a random subset of rows) and Feature subsampling (each split considers only √n features). This prevents trees from correlating, making the ensemble much better than any single tree. 
from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor

model = RandomForestClassifier(
    n_estimators=100,       # number of trees (more = better, but slower)
    max_depth=None,         # let trees grow fully (bagging handles overfitting)
    max_features='sqrt',   # features per split: sqrt(n) for classification
    min_samples_split=2,
    n_jobs=-1,              # use all CPU cores
    random_state=42
)
model.fit(X_train, y_train)
pred = model.predict(X_test)

# ── Feature importance ── (very useful for EDA!)
feat_imp = pd.Series(model.feature_importances_, index=X.columns)
feat_imp.nlargest(10).plot(kind='barh')  # top 10 features

Support Vector Machine (SVM)
Find the hyperplane that maximizes the margin between classes

🔑 Key Concepts

  • Hyperplane: the decision boundary (line in 2D, plane in 3D)
  • Margin: distance between hyperplane and nearest points. SVM maximizes this
  • Support Vectors: the data points closest to the hyperplane — they define it
  • Kernel Trick: maps data to higher dimensions for non-linear separation

🧪 Kernel Types

  • linear: for linearly separable data
  • rbf (Gaussian): general-purpose, most common default
  • poly: polynomial decision boundary
  • C parameter: higher C = harder margin (less tolerance for misclassification)
from sklearn.svm import SVC, SVR

model = SVC(
    kernel='rbf',      # linear, rbf, poly, sigmoid
    C=1.0,            # regularization: high=less regularization
    gamma='scale',    # rbf bandwidth: 'scale', 'auto', or float
    probability=True  # enable predict_proba() — slower
)
model.fit(X_train, y_train)
# Always scale features before SVM — very sensitive to magnitude

Clustering & KMeans
Unsupervised learning — find hidden structure in unlabeled data
ℹ️ KMeans minimizes inertia (within-cluster sum of squares). Use the Elbow Method: plot inertia vs K and find where the curve bends — that's your optimal K. 
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

# ── Elbow method to find best K ──────────────
inertias = []
for k in range(1, 11):
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X)
    inertias.append(km.inertia_)

plt.plot(range(1,11), inertias, 'bo-')
plt.xlabel('K'); plt.ylabel('Inertia')

# ── Fit KMeans ───────────────────────────────
model = KMeans(n_clusters=3, random_state=42, n_init=10)
model.fit(X)
labels   = model.labels_          # cluster assignment per sample
centers  = model.cluster_centers_  # centroid coordinates

# ── Silhouette score: how well separated clusters are ─
sil = silhouette_score(X, labels)   # 1 = perfect, -1 = wrong clusters

PCA — Principal Component Analysis
Reduce dimensionality while retaining maximum variance
⚠️ Always StandardScale before PCA. PCA is variance-based — large-scale features dominate the components if not scaled. Also: PCA components are not interpretable (they're linear combinations of original features). 
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# ── Step 1: Scale ────────────────────────────
X_scaled = StandardScaler().fit_transform(X)

# ── Step 2: Find how many components to keep ─
pca_full = PCA()
pca_full.fit(X_scaled)
explained = pca_full.explained_variance_ratio_.cumsum()
# Find n where explained variance >= 95%
n = (explained >= 0.95).argmax() + 1

# ── Step 3: Reduce dimensions ────────────────
pca = PCA(n_components=n)
X_reduced = pca.fit_transform(X_scaled)
print(f"Reduced {X.shape[1]} features → {n} components")

# ── Step 4: Visualize (2D) ───────────────────
pca2d = PCA(n_components=2)
X_2d  = pca2d.fit_transform(X_scaled)
plt.scatter(X_2d[:,0], X_2d[:,1], c=y, cmap='viridis')

Pipeline
Chain preprocessing and modeling steps — prevents leakage, simplifies code
💡 Using Pipeline is the professional way to build ML models. It ensures preprocessing is applied correctly in cross-validation, prevents data leakage, and makes deployment simple (one .pkl file handles everything). 
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.impute import SimpleImputer
from sklearn.ensemble import RandomForestClassifier

# ── Define feature groups ────────────────────
num_features = ['Age', 'Salary']
cat_features  = ['Gender', 'City']

# ── Numeric pipeline ─────────────────────────
num_pipe = Pipeline([
    ('impute', SimpleImputer(strategy='median')),
    ('scale',  StandardScaler())
])

# ── Categorical pipeline ──────────────────────
cat_pipe = Pipeline([
    ('impute', SimpleImputer(strategy='most_frequent')),
    ('encode', OneHotEncoder(handle_unknown='ignore'))
])

# ── Combine with ColumnTransformer ────────────
preprocessor = ColumnTransformer([
    ('num', num_pipe, num_features),
    ('cat', cat_pipe, cat_features)
])

# ── Full pipeline ─────────────────────────────
full_pipe = Pipeline([
    ('prep',  preprocessor),
    ('model', RandomForestClassifier(n_estimators=100))
])

full_pipe.fit(X_train, y_train)
pred = full_pipe.predict(X_test)

Cross Validation
Reliable model evaluation using multiple train-test splits
ℹ️ A single train-test split can be lucky or unlucky depending on which samples end up in each set. Cross-validation gives you a much more reliable estimate of true model performance by averaging across K different splits. 
from sklearn.model_selection import (
    cross_val_score, KFold, StratifiedKFold, cross_validate
)
import numpy as np

# ── Basic K-Fold ─────────────────────────────
scores = cross_val_score(model, X, y,
                          cv=5,           # 5 folds
                          scoring='accuracy')
print(f"CV Accuracy: {scores.mean():.3f} ± {scores.std():.3f}")

# ── Stratified K-Fold (recommended for classification)
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=skf)

# ── Get multiple metrics at once ─────────────
results = cross_validate(model, X, y, cv=5,
    scoring=['accuracy', 'f1', 'roc_auc'],
    return_train_score=True)
# results['test_accuracy'], results['train_accuracy']  (detect overfitting!)

Hyperparameter Tuning
Systematically find the best model settings

🔲 GridSearchCV

Tries every combination of parameters. Thorough but slow. Use when parameter space is small.

🎲 RandomizedSearchCV

Tries n_iter random combinations. Much faster. Often finds near-optimal results with 20-50 iterations.

from sklearn.model_selection import GridSearchCV, RandomizedSearchCV
from scipy.stats import randint, uniform

# ── GridSearchCV ─────────────────────────────
param_grid = {
    'n_estimators': [50, 100, 200],
    'max_depth'    : [None, 5, 10],
    'min_samples_split': [2, 5]
}  # 3×3×2 = 18 combinations × 5 folds = 90 fits

gs = GridSearchCV(
    RandomForestClassifier(), param_grid,
    cv=5, scoring='f1', n_jobs=-1, verbose=1
)
gs.fit(X_train, y_train)
print(gs.best_params_, gs.best_score_)

param_dist = {
    'n_estimators'    : randint(50, 500),

    'min_samples_split': randint(2, 20),

Complete ML Workflow
The end-to-end process for every project
  1. Define the problem — What are you predicting? Classification or Regression? What metric matters?
  2. Collect data — CSV, database, API, scraping. More quality data usually beats better algorithms.
  3. EDA — Distribution, correlations, outliers, class balance. Understand before you model.
  4. Preprocessing — Handle missing values, encode categoricals, scale numeric features.
  5. Feature Engineering — Create new useful features, remove irrelevant ones.
  6. Train/Test Split — Stratified 80/20 or 70/30. Hold out test set — don't touch it until final eval.
  7. Baseline model — Start simple (LogisticRegression, DummyClassifier). Know the floor to beat.
  8. Train and evaluate — Use cross-validation. Compare multiple algorithms.
  9. Tune hyperparameters — RandomizedSearchCV first, then GridSearchCV to refine.
  10. Final evaluation — Only now use the test set. Report metrics honestly.
  11. Save model — Save the full pipeline with joblib.
  12. Deploy — Flask/FastAPI REST API, or Streamlit app for demos.
